# 世界杯预测评测

6 家旗舰模型读 `team_data` 的 summary 做预测,赛后对照真实赛果打分 → 前端积分榜。

## 🚨 两条铁律

1. **开跑前必先汇报**:真正调模型跑测评(烧 DMX API)前,**必须先向用户汇报**(测什么 / 哪 6 模型 / 调用量·成本 / 防泄露确认),**等用户确认才跑**。见记忆 `feedback-eval-report-first`。
2. **防泄露**:只喂「该预测点**之前**」的干净快照——赛前全量=赛前 base/最新版;逐日=当轮快照(`snapshot_round` 出的)。**绝不把待预测的赛果喂进去。**

## 评测对象(`wc_eval/predict/models.json`,实测可调通)

Claude `claude-opus-4-8` · GPT `gpt-5.2` · Gemini `gemini-3.1-pro-preview` · Kimi `kimi-k2.5` · GLM `glm-5` · Seed `doubao-seed-2-0-pro-260215`。统一经 `wc_eval/llm_client.py`(DMXAPI)调。

## 三类预测

| 类 | 喂什么 | 问什么 | 脚本 |
|---|---|---|---|
| **A 单场 7 市场** | 对阵 **2 队完整 summary** | 胜平负 / 让球 / 大小2.5 / 双方进球 / 单双 / 半全场 / 正确比分 | `predict_match.py` |
| **B 小组头名** | 该组 **4 队完整 summary**（12 组各跑 1 次） | 该组头名(第 1) | `predict_group.py` |
| **C 全局彩池** | **48 队速览**(`common.extract_brief`:留 速览+⑦+⑧+市值、**去⑤阵容**5000字细节,~2000字/队 → 48 队≈127k token,放得进 200k 模型) | 夺冠 / 进决赛(2) / 四强(4) / 夺冠大洲 / 总进球大小(285.5) | `predict_global.py` |

> 喂法规律:**单场/头名喂完整 summary**(2~4 队放得下);**全局彩池必须喂速览**(48 队完整会 30 万字超 context,故 `extract_brief` 抽球队层面几段)。

## 评分(`score.py`,赛后对照真实赛果)

全局彩池(前端分值):夺冠 **+10**、进决赛 **+3/队**、四强 **+2/队**、夺冠大洲 **+2**、总进球 **+2**;
单场各市场 / 小组头名分值【待定·按前端结算规则填】。累加 = 单场分 + 全局分 → 积分榜。

## 流程

```
① 出题(从 bg/static/groups + 赛程 matches)
② 对每个模型、每道题：读相关队 summary → 构造 prompt → llm_client.chat → 解析结构化答案
③ 存预测 JSON（不评分,赛果未出）
④ 赛后：score.py 用真实赛果对照 → 打分 → 积分榜
```

## prompt 原则

- 把相关队 summary **原样**放进 system/user;明确「只依据给定资料预测、不许编」;
- 要求**结构化 JSON 输出**(便于解析评分),给定字段;
- 防泄露:summary 已是赛前/当轮干净版,prompt 里不另带任何赛果。

## 实测经验(2026-06-10 · 第一波 210 条已跑通)

- **prompt**:都「**先分析、再在最后输出 JSON**」(SOTA 自带 thinking,引导更稳;分析全文存进 `_raw` 便于人看);选项**能列全的列全**(半全场 9 种、让球档全列),免得各模型写法不一;
- **并行**:6 模型 / 多组用 `ThreadPoolExecutor`(max_workers 6-8);**`ask_json` timeout 调到 300s**(thinking 慢,**GLM 尤甚、偶 timeout 需重试**);
- **存储**:`predictions/<日期_时分>/`(精确到分钟、同一波同批次)+ `_meta.md`;`save_pred` **合并写**(补跑单模型不覆盖其他已成功的);
- **解析**:`ask_json` 抓**最后一个** JSON(容纳前面分析文字),返回 `{_json, _raw}`;
- **第一波(批次 `2026-06-10_1437`)**:单场 84 + 头名 72 + 全局 54 = **210 条**,6 模型全成功(喂 1310 赛前快照、防泄露);
- **待办**:赛果出后 `score.py` 对照打分(评分前清洗个别带括号的值,如 GPT「单双」抄了选项说明);6/12 调 snapshot 补当天场。
