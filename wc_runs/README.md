# wc_runs · 世界杯运行数据 & 产物

这届世界杯跑出来的**所有东西**:输入数据 + 流水线产物。**随采集和滚动预测逐步生长**,现在是空的,跑到哪步长哪步(类比你们的 `societybench_runs`)。

## 预期结构

| 子目录 | 是什么 | 何时生长 |
|---|---|---|
| `data/`   | 结构化硬数据:`teams.json`(48 队+分组)、`fixtures`(赛程)、`results`(赛果)、`odds`(赔率) | 开始采集时 |
| `raw/`    | 原始采集:`raw_web/`(48 队新闻)、`raw_media/`(48 队讨论) | 开始采集时 |
| `rounds/` | 滚动工作区,**每切一刀长一个**:`r0_pretournament/`、`r1_groupstage/`、`r2_round32/` … 每个里装 timeline + context + 题库 + 预测 + 成绩单 | 每个预测切点 |

## 滚动逻辑

`rounds/` 是世界杯区别于普通 event 的关键:普通 event 一次性出一套产物;世界杯**每个切点出一套**,最后能看到「赛前预测 vs 实际结果」「每轮预测的演变」——这正是滚动校准。
